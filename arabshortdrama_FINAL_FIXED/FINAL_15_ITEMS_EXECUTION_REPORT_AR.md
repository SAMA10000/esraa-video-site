# FINAL STRICT MYSQL REPAIR REPORT

This patch is PHP + MySQL only. No Bolt Database, no Supabase, no Deno runtime.

## Final remaining-list execution

1. Bot messages are now generated by `/api/public/bot-message.php`, inserted into MySQL `chat_messages` with `is_bot=1`, and also inserted into `bot_message_logs`.
2. Bot moderation is available through existing chat moderation actions: `reviewChatMessage`, `hideChatMessage`, `deleteChatMessage` for both visitor and bot rows.
3. Bot/offer wording related to partner/sponsor/activation was replaced with neutral wording such as المتجر / الخطوة / التأكيد / فتح المنصة.
4. Preview bootstrap admin is implemented only for non-production preview hosts when `ALLOW_PREVIEW_BOOTSTRAP_ADMIN=true`; it is blocked on `arabshortdrama.cloud`.
5. `DENO_DEPLOYMENT_ID` is explicitly not required because the backend is PHP + MySQL, and the login debug panel reports that.
6. Admin login debug panel now reports endpoint, method, backend type, HTTP status, JSON status, CORS/same-origin status, and safe server debug if present.
7. Backend moderation is now intentionally mandatory; the fake toggle is removed from UI and backend forces it to enabled to keep links/phones blocked.
8. Storage Security Audit now includes source-level known keys and runtime unknown keys.
9. First-free access moved from `localStorage` to MySQL-backed `/api/public/gate-access.php` with HttpOnly viewer cookie and `viewer_series_access` table.
10. CTA code remains a real outer `<button type="button">`; live click tests must still be done in a real browser after deploy.
11. Related-video fallback now shows a real button "تشغيل مقطع مشابه" using same-category selection.
12. Deploy script removes old physical admin folders and runs route curl checks after deployment.
13. System health now includes endpoint file checks, bot-message endpoint, gate endpoint, viewer table, bot log table, and last safe backend error.
14. System health is still a backend smoke/file/table check; full browser UI testing remains manual after deploy.
15. Manual acceptance proof is produced by deploy smoke output where possible; browser click/Bolt badge tests remain impossible without opening the site.
